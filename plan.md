# WolfNote 狼人杀笔记软件开发计划

## 项目背景
针对狼人杀对局中信息量大、记录混乱的问题，开发一款结构化的笔记应用。采用自定义标记语言输入，提供多维度的视图（人员视图、身份总览、身份分线）来辅助逻辑推导。

## 技术栈
- **前端框架**: React + Vite (高效、轻量)
- **样式**: Tailwind CSS (移动优先，快速构建 UI)
- **状态管理**: React Context / Hooks (管理全局对局状态)
- **呈现形式**: PWA (渐进式 Web 应用，可在手机上像 App 一样使用)

## 功能模块

### 1. 新开一局 (Setup)
- 设置玩家人数 (6-18人)
- 配置身份配置：预设普通村民、预言家、女巫、猎人、守卫、白痴、狼人、倒钩狼人等。
- 自定义各身份数量。

### 2. 标记语言解析引擎 (Markup Parser)
- `v[target]:[voters]` - 投票信息。例如 `v3:1,2` (1,2号投3号)。
- `v0:[voter]` - 弃票。例如 `v0:5` (5号弃票)。
- `b[player]:[role][weight]` - 身份标记。例如 `b1:预言家` 或 `b1:女巫0.5`。
- `s[player]:[status]` - 状态标记（如倒钩、冲锋）。例如 `s1:倒钩`。
- `t[player]:[content]` - 发言记录。例如 `t1:自认女巫，踩3`。
- 区分警长竞选和普通投票。

### 3. 多视图系统
- **主页面 (Log View)**: 时间线形式展示所有输入的原始记录，带解析后的可视化效果。
- **人员视图 (Player View)**: 选择特定玩家，聚合其所有的投票、被投、身份标记和发言。
- **身份总览 (Role Matrix)**: 表格/矩阵形式展示当前所有玩家的身份嫌疑度和标记。
- **身份分线 (Hypothesis/Branches)**:
    - 允许插入“如果 A 是真预言家”的分支。
    - 在分支下记录逻辑（B, C 是狼）。
    - 快速切换分支进行对比。

## 开发阶段
1. **Phase 1**: 环境搭建与基础数据结构设计。
2. **Phase 2**: 解析引擎开发。
3. **Phase 3**: 核心 UI 开发（设置页、输入页）。
4. **Phase 4**: 高级视图开发（人员、总览、分线）。
5. **Phase 5**: 移动端适配与 PWA 配置。

## 验证方法
- 模拟标准狼人杀对局（如 12 人预女猎守局）进行全流程输入测试。
- 验证标记语言在各种边界情况下的解析准确性。
- 验证身份分线在复杂逻辑推导下的可用性。
